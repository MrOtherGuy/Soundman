Control media playback

Soundman will create a new context menu item for each tab that plays audio. You can see the tab title and control that tab directly from that menu.

Soundman requires very broad permissions to all tabs and websites because it sends a script to webpage when you try to pause/play media in that tab.

Instructions

Use configurable click modifiers to take different actions when you click the context menu entry. Defaults actions are:

  • No modifier: Switch to the tab
  • Shift: Mute/Unmute the tab
  • Ctrl: Try to play/pause the playback

When you use Soundman context menus to mute or pause playback the tab will stay in context menu and you can later use it to resume playback.

You can pin a tab to Soundman by using the relevant menu entry when you right-click a tab. This way you can pin a tab to context menu even if that tab plays no audio.

When you switch to tab using Soundman you can switch back to the original tab using the same menu again.

Menuitems are removed when:

  • The tab is closed
  • You stop the playback from within the tab (if the tab is not pinned)
  • Playback stops on it's own (if the tab is not pinned)
  • You remove the tab from Soundman via tab context menu

Known issues

  • Playback control (play/pause) is website dependent and may not work everywhere and might behave unexpectedly when there are multiple audio sources. Youtube, Soundcloud, Mixcloud, Netflix work at the time of writing.
  • Soundman tab states don't persist between Firefox restarts
